# 朝夕：项目事实与产品约定

这份文件是后续开发任务的“唯一事实来源”。开始修改前先读它，避免让用户在新对话里重复背景和已经确定的选择。

## 1. 产品定位

- 产品名：朝夕。
- 用户名：29。
- 定位：自己使用的轻量监督教练，不做复杂健康管理，也不做泛用 AI 聊天。
- 视觉：米黄色与白色为主，简约、温和、克制、有高级感。
- 主导航固定为：打卡、日历、周报、我的。

## 2. 打卡

- 用户可以添加和编辑项目：名称、每日目标、单位、提醒时间、项目头像。
- 手机端左滑项目卡片后显示“设置”和“删除”。桌面端按钮可以直接看见。
- “删除”实际为归档：从当前打卡列表移除，但历史日志和日历明细必须保留。
- 目标大于 1 时可逐次记录，并可撤销；达到目标后显示完成。
- APK 使用原生本地通知；网页不承诺可靠的后台定时提醒。

## 3. 日历

- 月视图从星期日到星期六排列。
- 每天显示日期和完成百分比，并按完成率加深颜色。
- 在日历网格向上滑进入下个月，向下滑进入上个月；也保留左右月份按钮。
- 点选某天显示项目明细，归档项目有历史记录时仍要显示。
- 新增或归档项目不能篡改它生效日期之外的历史完成率。

## 4. AI 周报

- 周期固定为周六至周五；周五 23:57 触发本地提醒。
- 输入包含最近一个完整周期的逐日客观数据，以及此前四周每个项目的完成天数和完成率。
- 输出只做四件事：本周亮点、需要注意、可能忽略的规律、下周建议。
- 必须引用已有项目和数字；数据不足时明确说明，禁止编造。
- 禁止健康诊断、医疗建议和与这份周报无关的泛用聊天。
- 每个周期缓存一份正式周报；为处理偶发低质量结果，最多允许生成两次。
- 每份周报最多追问五次，追问只能围绕该周报和对应统计。
- 模型：硅基流动 `deepseek-ai/DeepSeek-V4-Pro`。
- API Key 绝不提交到 Git、网页或 APK；数据备份也必须排除 API Key。
- 离线版的打卡、日历、历史周报和数据管理不依赖 AI 或网络；AI 不可用时不得影响应用启动。

## 5. 数据与隐私

- 项目、日志、头像、周报和用户设置默认只存在当前设备的 IndexedDB。
- 支持完整 JSON 导出、导入和清空。
- AI 请求只发送周报所需的紧凑统计、当前周报和有限追问，不发送头像或无关数据。
- 目前没有账号同步；更换设备前需手动导出备份。

## 6. 网页、托管与 APK

- 正式下载页优先使用当前可从中国大陆访问的 EdgeOne Makers 免费项目，不购买付费套餐。
- GitHub 用于源代码备份和版本管理；纯 GitHub Pages 在中国大陆晚上高峰期访问不稳定，不作为正式托管。
- 正式网站需要尽可能让中国大陆用户无需代理即可访问。
- APK 是 Capacitor 原生外壳，但完整网页资源必须内置在安装包中，启动不依赖任何远程网址。
- 离线版使用固定应用编号 `com.shihao29.zhaoxi` 和固定签名；未来更新只允许提高版本号并覆盖安装。
- 离线版与旧联网版使用不同应用编号，首次迁移时可以并存。用户需先导出旧版数据、在离线版导入并核对，再删除旧版。
- APK 内不能出现“获取安卓版”、下载 APK、安装步骤等网页专属内容。
- 正式网页只作为 APK 下载入口，打开后自动开始下载，并保留手动下载按钮作为兼容兜底。

## 7. 交付验收

- Lint、应用测试、Cloudflare Pages 的 Next.js 构建均通过。
- 在移动端尺寸完成至少两轮体验检查，覆盖添加、编辑、头像、左滑、归档、打卡、日历、导入导出、周报限额。
- 线上验证首页、AI 接口和 APK 下载均可用。
- 断网验证最终 APK 可以启动，且 APK 不包含远程 `server.url` 或任何 API Key。
- 使用相同签名完成一次覆盖升级验证，并验证 IndexedDB 数据仍然存在。
- 完成源码提交与 GitHub 备份后再宣告交付完成。
